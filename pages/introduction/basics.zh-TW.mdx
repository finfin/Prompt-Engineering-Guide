# 提示基礎

## 基本提示

提示可以做到許多事，但結果的品質取決於您提供的訊息量以及提示的品質。提示可以包含像 *指示* 或 *問題* 等訊息，並包括其他細節，如 *內文*、*輸入* 或 *範例*。您可以使用這些要素更好地指導模型，從而獲得更好的結果。

讓我們先看一個簡單的基本範例：

*提示*
```
天空是
```

*輸出*
```
天空是藍色的，這是因為大氣中的氣體和其他微小顆粒物質散射陽光。當太陽光穿過大氣層時，短波長的藍光比長波長的紅光更容易被散射。因此，當我們看向天空時，主要看到的是藍光。然而，在不同的時間和天氣條件下，天空的顏色可能會有所變化，例如在日出和日落時，天空可能呈現出紅、橙、黃等顏色。
```


如您所見，當給了`"天空是"`的內文後，語言模型輸出了接續的字串。但輸出可能出乎意料或者與您想完成的任務相去甚遠。

透過這個基本範例，可以看到提供更多內文或具體說明您想要達成的目標的必要性。

讓我們試著改進一下：

*提示：*
```
完成句子：

天空是
```

*輸出：*
```
天空是遼闊無垠的，擁有千變萬化的色彩和形態，既是大自然的藝術品，也是人類夢想和探索的舞台。
```


有改善嗎？您告訴模型完成句子，所以結果看起來好多了，因為它完全按照您的要求（“完成句子”）來做。這種設計並且最佳化提示來讓模型確切執行任務的方法被稱為**提示工程**。

以上範例是一個關於當今 LLM 能力的基本說明。當今的 LLM 能夠執行各種高級任務，包括文本摘要、數學推理和程式碼生成。

## 提示格式

您已經嘗試了一個非常簡單的提示。標準提示的格式如下：

```
<問題>?
```

或
```
<指令>
```

您可以將其格式化為問答（QA）格式，這在許多 QA 資料集中是標準的，如下所示：

```
Q: <問題>?
A: 
```

像上面這樣提示時，也被稱為 *零樣本提示* ，即您直接要求模型回應，而不提供任何關於您希望其實現的任務的範例或示範。一些大型語言模型確實具有執行零樣本提示的能力，但這取決於手頭任務的複雜性和知識。

考慮到上面的標準格式，一種流行且有效的提示技術被稱為 *少樣本提示* ，您可以提供範例（即示範）。您可以按照以下格式進行少樣本提示：

```
<範例問題>?
<範例回答>

<範例問題>?
<範例回答>

<範例問題>?
<範例回答>

<問題>?

```

QA 格式版本如下：

```
Q: <範例問題>?
A: <範例回答>

Q: <範例問題>?
A: <範例回答>

Q: <範例問題>?
A: <範例回答>

Q: <問題>?
A:
```

請記住，使用 QA 格式並非必需。提示格式取決於手頭的任務。例如，您可以執行一個簡單的分類任務並給出示範該任務的範例，如下所示：

*提示：*
```
這太棒了! // 正面
這好爛! // 負面
電影很讚! // 正面
這表演慘不忍睹! //
```

*輸出：*
```
負面
```

少樣本提示可以實現內文中的學習，這是語言模型在給定一些示範後學習任務的能力。
