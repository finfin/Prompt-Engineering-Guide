# 設計提示的技巧

在設計提示時，注意以下幾點：

### 從簡單開始
在設計提示時，您應該記住，這實際上是一個迭代過程，需要大量的實驗才能獲得最佳結果。使用 OpenAI 或 Cohere 的 playground 是一個好的起點。

您可以從簡單的提示開始，並在追求更好的結果時不斷添加更多要素和內文。因此，隨時迭代提示至關重要。在閱讀指南時，您將看到許多範例，其中特定性、簡單性和簡潔性通常會為您帶來更好的結果。

當您有一個涉及多個不同子任務的大任務時，您可以嘗試將任務分解為更簡單的子任務，先讓子任務產生更好的結果後再繼續。這避免了在一開始的提示設計過程中添加過多的複雜性。

### 指令
您可以使用指令來設計各種簡單任務的有效提示，例如 "撰寫"、"分類"、"摘要"、"翻譯"、"排序" 等，以指示模型要實現的目標。

請記住，您還需要進行大量實驗，以了解哪種方式最有效。嘗試使用不同的指令、關鍵字、內文和資料，看看哪些對您的特定案例和任務最有效。通常，內文與您要執行的任務越具體和相關，效果就越好。我們將在即將到來的指南中談到取樣和添加更多內文的重要性。

另外建議您將指令放在提示的開頭。另一個建議是使用像 "###" 這樣的清晰分隔符來分隔指令和內文。

例如：

*提示：*
```
### 指令 ###
將以下文字翻譯成西班牙語：

哈嘍！
```

*輸出：*
```
¡Hola!
```


### 特定性
對於您希望模型執行的指令和任務要非常具體。提示越具描述性和詳細，結果就越好。這在尋求特定結果或生成風格時尤其重要。並非特定的標記或關鍵字會帶來更好的結果。更重要的是擁有良好的格式和描述性提示。事實上，在提示中提供範例對於以特定格式獲得所需輸出非常有效。

在設計提示時，您還應注意提示的長度，因為提示長度有一定的限制。思考一下您應該多具體和詳細。包含太多不必要的細節並不一定是好方法。細節應該與任務相關並有助於完成任務。這需要您進行大量實驗。我們鼓勵大量的實驗和迭代，以優化您的應用程序的提示。

作為範例，讓我們嘗試一個簡單的提示，從文本中提取特定資訊。

*提示：*
```
從以下文本中提取地名。

期望的格式：
地點：<以逗號分隔的列表>

輸入：儘管這些進展對研究人員來說是令人鼓舞的，但仍有很多未知之謎。里斯本查普利莫德未知中心的神經免疫學家亨里克·維加-費爾南德斯（Henrique Veiga-Fernandes）說："我們經常在大腦和我們在外周看到的效果之間有一個黑盒子。" "如果我們想在治療背景下使用它，實際上我們需要了解該機制。"
```

*輸出：*
```
地點：里斯本查普利莫德未知中心
```

輸入文本取自[這篇 Nature 文章](https://www.nature.com/articles/d41586-023-00509-z)。

###避免不精確

遵照上述關於詳細和改進格式的技巧，很容易陷入提示過於巧妙並可能產生不精確描述的陷阱。更好的方法通常是具體和直接。這裡的類比非常類似於有效的溝通——越直接，信息傳達得越有效。

例如，您可能對了解提示工程的概念感興趣。您可以嘗試：

```
解釋提示工程的概念。將解釋保持簡短，僅需幾句話，不要過於詳細。
```

從上述提示中無法清楚地了解要使用多少句子以及風格。您可以用上述提示獲得一定程度的結果，但要若改成具體、簡潔且直接的提示可以得到更好的效果。比如:

```
用兩到三句話向一位高中生解釋提示工程的概念。
```

### 做還是不做？
在設計提示時的另一個常見建議是避免說明不要做什麼，而應該說明要做什麼。這鼓勵更具體性，並專注於使模型產生良好回應的細節。

以下是一個電影推薦聊天機器人在我編寫指令時未能達到我不希望它做的事情的例子，因為我專注於不要做什麼。

*提示：*
```
以下是一個向顧客推薦電影的代理商。不要詢問興趣。不要詢問個人資訊。

顧客：請根據我的興趣推薦一部電影。
代理商： 
```

*輸出：*
```
當然，我可以根據您的興趣為您推薦一部電影。您想看哪種類型的電影？您喜歡動作片、喜劇、愛情片還是其他什麼？
```

這是一個更好的提示：

*提示：*
```
以下是一個向顧客推薦電影的代理商。代理商負責從全球熱門電影中推薦一部電影。它應避免詢問用戶偏好，並避免詢問個人資訊。如果代理商無法推薦一部電影，應回覆 "對不起，今天找不到適合推薦的電影。"。

顧客：請根據我的興趣推薦一部電影。
代理商：
```

*輸出：*
```
對不起，我不了解您的興趣。然而，以下是目前全球熱門電影的清單：[電影清單]。希望您能找到喜歡的！
```

(ps. 在 GTP 3.5 之後模型已經可以精確處理上述的範例，但『著重說明要做什麼，避免說明不要做什麼』這個概念仍然適用)

以上範例來自[使用 OpenAI API 進行提示工程的最佳實踐"文章](https://help.openai.com/en/articles/6654000-best-practices-for-prompt-engineering-with-openai-api)。
